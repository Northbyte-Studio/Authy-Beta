# Security Policy

# Supported Versions
Authy is currently in beta, and security updates are applied only to the most recent development branch. Older versions and unofficial distributions are not supported.

# Reporting a Vulnerability
Security is a core priority for Authy. If you discover a vulnerability, please report it responsibly so it can be addressed quickly and safely. When reporting a security issue, please include the following information:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact (e.g., data exposure, crash, bypass)
- Device model and Android version
- Any logs or screenshots that help illustrate the problem

# How to Report
Until the official website is online, please report vulnerabilities through the following channels:

- GitHub Issues (mark them as Security)
- Direct contact with Northbyte Studios (contact details will be provided once the website is restored)

Please refrain from publicly disclosing vulnerabilities until they have been reviewed and addressed.

# Security Expectations
Authy handles sensitive information such as TOTP secrets, passwords, and private notes. To maintain user safety, please adhere to the following guidelines:

- Do not submit malicious code or proofs of concept that could harm devices.
- Do not attempt to exploit other users or distribute compromised builds.
- Do not share sensitive data or private keys in issues or pull requests.

All contributions must follow secure coding practices and respect user privacy.

# Vulnerability Handling Process
When a report is received, the following steps will be taken:

1. The issue will be reviewed and validated.
2. A fix will be developed and tested.
3. A patched version will be released.
4. A security note will be added to the changelog.

Critical issues will be prioritized and addressed as quickly as possible.

# Responsible Disclosure
Northbyte Studios supports responsible disclosure. Publicly posting vulnerabilities before they are fixed may put users at risk. Please allow time for investigation and patching before sharing any details.
