# Contributing to Authy
Thank you for your interest in contributing to Authy, developed by Northbyte Studios. Authy is currently in beta, and community feedback is essential for improving stability, security, and usability. This document outlines how to contribute effectively to the project.

# How to Contribute
## Reporting Issues
If you encounter bugs, crashes, UI problems, or unexpected behavior, please open an issue in the repository. Include as much detail as possible:
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Device model and Android version
- Screenshots or logs (if available)
Clear reports help us diagnose and resolve problems more quickly.
## Suggesting Features
Feature requests are welcome. When submitting a suggestion:
- Describe the feature and its purpose
- Explain why it would benefit users
- Include examples or mockups if relevant
We review all suggestions and prioritize them based on feasibility and user impact.

## Development Status
Authy is in active beta development. The APK is currently too large to host directly on GitHub, and the official website for downloads and documentation is under construction. Once the site is live, contributors will have access to builds, changelogs, and additional resources.

## Code of Conduct
All contributors are expected to interact respectfully and constructively. Harassment, discrimination, or disruptive behavior will not be tolerated. A full Code of Conduct will be added as the project expands.
